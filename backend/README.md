# Rowaq App Backend

Backend server for Rowaq App with MongoDB database and REST API.

## Prerequisites

- Node.js (v16 or higher)
- MongoDB (local or Atlas)
- npm or yarn

## Installation

1. Navigate to backend directory:
```bash
cd backend
```

2. Install dependencies:
```bash
npm install
```

3. Create `.env` file with your configuration:
```env
PORT=5000
MONGODB_URI=mongodb://localhost:27017/rowaq-app
NODE_ENV=development
JWT_SECRET=your_secret_key
```

## Running the Server

### Development Mode (with auto-reload):
```bash
npm run dev
```

### Production Mode:
```bash
npm start
```

The server will start on `http://localhost:5000`

## API Endpoints

### Managers
- `GET /api/managers` - Get all managers
- `GET /api/managers/:id` - Get manager by ID
- `POST /api/managers` - Create new manager
- `PUT /api/managers/:id` - Update manager
- `DELETE /api/managers/:id` - Delete manager
- `POST /api/managers/bulk-import` - Bulk import managers

## Database Setup

### Using Local MongoDB

1. Install MongoDB from https://www.mongodb.com/try/download/community

2. Start MongoDB:
```bash
mongod
```

### Using MongoDB Atlas (Cloud)

1. Create account at https://www.mongodb.com/cloud/atlas
2. Create a cluster and get connection string
3. Update `.env` file:
```env
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/rowaq-app
```

## Frontend Configuration

Update `.env` in the frontend root directory:
```env
VITE_API_URL=http://localhost:5000/api
```

Or the frontend will automatically use `/api` for development due to Vite proxy configuration.

## Notes

- All timestamps are automatically added by MongoDB
- IDs are automatically generated by MongoDB (_id field)
- Data validation is handled at the model level
- Errors are returned as JSON with appropriate HTTP status codes
